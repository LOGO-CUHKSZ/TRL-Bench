# Table Retrieval

## Overview

Given a natural language question, retrieve the relevant table from a corpus of Wikipedia tables.

Reference: NQ-Tables dataset (derived from Natural Questions)

## Embeddings Consumed

> **Embedding Level:** Table
> **Primary Embedding:** `table_embedding` or `cls_embedding`
> **Query Embeddings:** Always generated by BERT (the text encoder)
> **Retrieval Modes:** Hybrid (recommended) or Model-only (baseline)

### Query Embeddings (Always BERT)

Query embeddings are always generated by BERT. They live at:
`embeddings/table_retrieval/bert/queries_{split}.pkl`

### Table Embeddings

Table embeddings can come from any table encoder (BERT, TaBERT, TAPAS, TabSketchFM, etc.).
Two retrieval modes are supported for non-BERT encoders:

- **Hybrid (recommended)**: Combine the encoder's table embeddings with BERT table
  embeddings (concatenation) using `create_hybrid_embeddings.py`. This bridges the
  modality gap between the BERT query space and the table embedding space.
- **Model-only (baseline)**: Use the encoder's raw table embeddings directly. A learned
  projection head aligns the model's table space to the BERT query space. This measures
  the marginal alignment capability of each model's embeddings without BERT table assistance.

| Embedding Type | Required | Shape | Description | Compatible Models |
|----------------|----------|-------|-------------|-------------------|
| `table_embedding` | **Yes** | `dict (v2.0)` | Table-level embeddings | TaBERT, TAPAS, TabSketchFM, Doduo, TURL |
| `column_embeddings` | Optional | `{col_idx: (dim,)}` | Per-column embeddings | All column models |

**Table embedding structure (v2.0):**
```python
'table_embedding': {
    'cls_embedding': array or None,  # [CLS] token (TAPAS, TabSketchFM)
    'table_embedding': None,         # No native support
    'column_mean': array,            # Mean-pooled columns
    'token_mean': array or None,     # Mean of all non-padding token hidden states
}
```

**Helper:** Use `get_table_level_embedding(item, variant='column_mean')` to extract the desired variant.

### Query Embeddings

| Embedding Type | Required | Shape | Description | Model |
|----------------|----------|-------|-------------|-------|
| Query embedding | **Yes** | `(dim,)` | Question embedding | **BERT** (always) |

## Supported Input Formats

### Format 1: Unified List Format v2.0 (Pickle) — Recommended

**Table embeddings** (from `embeddings/column/{model}/{dataset}.pkl`):
```python
[
    {
        'table_id': 'table_name',  # Canonical identifier (for ground truth matching)
        'table': 'path/to/table.csv',  # Full path to source file
        'column_embeddings': {0: [768], 1: [768], ...},
        'table_embedding': {
            'cls_embedding': array or None,
            'table_embedding': None,
            'column_mean': array,
            'token_mean': array or None,
        }
    },
    ...
]
```

**Hybrid embeddings** (from `create_hybrid_embeddings.py`) include provenance metadata:
```python
{
    'table_id': 'table_name',
    'table_embedding': {variant: hybrid_array},
    'hybrid_base_variant': 'token_mean',       # what was used from base encoder
    'hybrid_bert_variant': 'column_mean',       # what was used from BERT
    'hybrid_combination_method': 'concat',      # how they were combined
}
```

**Query embeddings** (always BERT, from `embeddings/table_retrieval/bert/queries_{split}.pkl`):
```python
[
    {
        'text_id': 'question_id',
        'text': 'What is...?',
        'embedding': array  # (dim,)
    },
    ...
]
```

**Note:** `table_id` is the preferred identifier. If not present, loaders extract it from the `table` path.

### Format 2: Dict Format (Pickle) — Deprecated

> **Note:** The legacy dict format is no longer supported by `load_table_embeddings()`.
> Regenerate embeddings in the unified list format (v2.0) above.

## Example Commands

### Training (Hybrid — Recommended)

**Variant semantics:**
- `--base_variant` is the **experiment knob** — selects which table-level variant to extract from the base encoder
- `--bert_variant` is the **task default** (`column_mean`) — only override for debugging or ablations
- Generated SLURM jobs always fix `bert_variant=column_mean`; only `base_variant` is swept

```bash
# Step 1: Create hybrid table embeddings (e.g. TAPAS + BERT)
# base_variant = experiment knob; bert_variant defaults to column_mean
python -m downstream_tasks.table_retrieval.create_hybrid_embeddings \
    --base_variant token_mean \
    --base_embeddings embeddings/column/tapas/nq_tables.pkl \
    --bert_embeddings embeddings/column/bert/nq_tables.pkl \
    --output_path embeddings/column/tapas_bert_hybrid/nq_tables.pkl

# Step 2: Train (query embeddings are always BERT)
python -m downstream_tasks.table_retrieval.train \
    --table_embeddings embeddings/column/tapas_bert_hybrid/nq_tables.pkl \
    --table_id_mapping datasets/nq_tables/csv/table_id_to_csv.json \
    --train_query_embeddings embeddings/table_retrieval/bert/queries_train.pkl \
    --train_questions datasets/nq_tables/json/train.json \
    --dev_query_embeddings embeddings/table_retrieval/bert/queries_dev.pkl \
    --dev_questions datasets/nq_tables/json/dev.json \
    --output_dir assets/checkpoints/table_retrieval/tapas_bert_hybrid
```

### Evaluation

```bash
python -m downstream_tasks.table_retrieval.evaluate \
    --table_embeddings embeddings/column/tapas_bert_hybrid/nq_tables.pkl \
    --table_id_mapping datasets/nq_tables/csv/table_id_to_csv.json \
    --query_embeddings embeddings/table_retrieval/bert/queries_dev.pkl \
    --questions_path datasets/nq_tables/json/dev.json \
    --projection_head assets/checkpoints/table_retrieval/tapas_bert_hybrid/best_model.pt \
    --output_path results/evaluation/table_retrieval/tapas_bert_hybrid_dev.json \
    --gpu
```

### Training (Model-only — Baseline)

```bash
# No hybrid creation needed — use raw model embeddings directly
python -m downstream_tasks.table_retrieval.train \
    --table_embeddings embeddings/column/tapas/nq_tables.pkl \
    --embedding_variant column_mean \
    --table_id_mapping datasets/nq_tables/csv/table_id_to_csv.json \
    --train_query_embeddings embeddings/table_retrieval/bert/queries_train.pkl \
    --train_questions datasets/nq_tables/json/train.json \
    --dev_query_embeddings embeddings/table_retrieval/bert/queries_dev.pkl \
    --dev_questions datasets/nq_tables/json/dev.json \
    --output_dir assets/checkpoints/table_retrieval/tapas_model_only
```

## Troubleshooting

### Common Issues

1. **Shape mismatch between tables and queries**
   - Ensure both use same embedding dimension
   - Check projection head input dimensions match

2. **Missing table_id in embeddings**
   - Verify table_id field exists in pkl entries
   - Check for table_id format consistency (e.g., with/without .csv extension)
